1 """
2 This module, `kcp_v` (Kernel Change Point Visualisation), includes a function for visualization and
3 explainability. It extracts intervals around each detected change point, illustrating the
4 contribution of each sensor to the change point criterion. Each interval is bounded on the left by
5 either 0 or the previous change point location and on the right by either the next change point or
6 the time series end-point. The visualization highlights the change point's location with a vertical
7 dotted red line and displays the algorithm criterion as a blue curve.
8  
9 Remarks:
10 1. For clarity in visualization, individual time series related to different change points are
11 visualized separately, resulting in one plot per detected change point.
12 2. The visualization function accommodates the algorithm's processing of a maximum of
13 "max_n_time_points" in each loop. If the total exceeds "max_n_time_points," the interval
14 calculation differs for the last change point found in each loop. Hence, "max_n_time_points" must
15 be consistent between the change point detection and visualization functions.
16 3. Only the linear kernel provides criterion lines for each individual time series that sum up to
17 the value of the global criterion line. Although designed for the linear kernel, the visualization
18 function can be used for other kernels (if the same kernel is used in the kcp_ds function).
19 4. #data: The input data array must be the exact same array used in the kcp_ds function.
20 """
